# Replit Agent Prompt — Fix `LoanType` Field (Use Deal-Analysis loan_program Instead)

## Context

The `LoanType` field in extraction results has been returning wrong values. Recent example: a Standard 7(a) loan was extracted as `"Variable"` because the model latched onto an `"Interest Type: Variable"` line in the document. Reading the v3 prompt, this is unsurprising: `LoanType` is listed in the schema dictionary but there are **no instructions anywhere** in the prompt explaining what `LoanType` means or what values are valid. Claude is guessing in the dark and citing a plausible-looking but wrong substring.

The root cause is that `LoanType` and `loan_program` are duplicates. The deal-analysis prompt already classifies every document as one of:

```
"SBA 7(a) Standard", "SBA 7(a) Express", "SBA 504", "Conventional"
```

…and the result is validated by `DealStructure` in `extraction/models.py` and stored on `deal.loan_program`. The field-extraction prompt should not be re-extracting this — it should just consume it.

This change does three things in one commit:

1. Remove `LoanType` from the field-extraction schema so the model stops guessing.
2. In the pipeline, populate `formatted_data["LoanType"]` from `deal["loan_program"]` after deal analysis.
3. Add a generic per-field allowed-values validator at the Pydantic boundary so any future field with a constrained value space will surface as a `schema_validation` failure instead of silently passing through.
4. Backfill existing rows in the database whose `formatted_data.LoanType` is wrong.

This is a v4 of the field-extraction prompt — do not edit v3. Existing extractions and v3 stay intact for audit.

## Step 5 — Generic per-field allowed-values validator

File: `artifacts/sba-backend/extraction/models.py`

Right now `validate_extracted_fields(raw, expected_keys)` accepts any string for any field. The `LoanType` bug slipped through partly because the validator had no concept of "this field has a constrained value space."

Add a module-level constant declaring allowed values for fields that have them:

```python
# Per-field allowed value sets. Fields not in this dict accept any string.
# Add new entries as the schema gains constrained-value fields.
ALLOWED_FIELD_VALUES: Dict[str, Set[str]] = {
    # Add future entries here. LoanType is now derived from deal_analysis
    # and is not extracted by the field-extraction prompt, so it does not
    # need an entry — but the deal-analysis stage's DealStructure model
    # already enforces loan_program via Literal[...] on the Pydantic side.
}
```

Leave the dict empty for now — `LoanType` doesn't belong here because it's no longer extracted. The point of adding the empty dict is so the validator infrastructure exists for the next time a constrained-value field is added.

Extend `validate_extracted_fields` to consult this dict. After the existing loop that drops extras and coerces non-strings, add a second pass:

```python
violations: list[str] = []
for k, v in cleaned.items():
    if not v:
        continue
    allowed = ALLOWED_FIELD_VALUES.get(k)
    if allowed is not None and v not in allowed:
        violations.append(f"{k}={v!r} (allowed: {sorted(allowed)})")

if violations:
    logger.error(
        "validate_extracted_fields: %d field(s) violated allowed-value constraints: %s",
        len(violations), violations,
    )
    raise ValueError(
        f"Field-extraction output contained values outside allowed sets: {violations}"
    )
```

The existing call site in `schemas.py:extract_fields` already wraps `validate_extracted_fields` in a try/except that converts exceptions to `ExtractionStageError` with `reason="schema_validation"`, so violations will surface through the existing degraded-extraction UX. No additional plumbing needed.

## Step 6 — One-shot backfill script for existing rows

Create `artifacts/sba-backend/scripts/backfill_loantype_from_deal.py`:

```python
"""
Backfill formatted_data.LoanType in existing extractions to mirror
deal_structure.loan_program. Run once after deploying the LoanType fix.

Idempotent: safe to re-run. Updates only rows where the current LoanType
differs from deal_structure.loan_program.
"""

import os
import sys
import json
from pathlib import Path

THIS_DIR = Path(__file__).resolve().parent
BACKEND_DIR = THIS_DIR.parent
if str(BACKEND_DIR) not in sys.path:
    sys.path.insert(0, str(BACKEND_DIR))

import db  # noqa: E402


def backfill() -> int:
    if not os.environ.get("DATABASE_URL"):
        raise RuntimeError("DATABASE_URL is not set; cannot run backfill.")

    updated = 0
    with db.get_cursor() as cur:
        cur.execute("""
            SELECT id, deal_structure, formatted_data, raw_data, field_sources
            FROM sba_extractions
        """)
        rows = cur.fetchall()

        for row in rows:
            deal = row["deal_structure"] or {}
            formatted = row["formatted_data"] or {}
            raw = row["raw_data"] or {}
            sources = row["field_sources"] or {}

            program = (deal.get("loan_program") or "").strip()
            if not program:
                continue
            current = (formatted.get("LoanType") or "").strip()
            if current == program:
                continue

            formatted["LoanType"] = program
            raw["LoanType"] = program
            sources["LoanType"] = {"quote": "[deal_analysis]", "verified": None}

            cur.execute("""
                UPDATE sba_extractions
                SET formatted_data = %s,
                    raw_data       = %s,
                    field_sources  = %s
                WHERE id = %s
            """, (
                json.dumps(formatted),
                json.dumps(raw),
                json.dumps(sources),
                row["id"],
            ))
            updated += 1

    return updated


if __name__ == "__main__":
    n = backfill()
    if n == 0:
        print("Backfill complete: all rows already aligned.")
    else:
        print(f"Backfill complete: corrected LoanType on {n} row(s).")
```

Run it once after deploying. The agent does not need to execute it — flag in the commit message that it should be run manually.
